Mentat-Learn

A self-improving personal assistant — one persona across every channel, a skill library that actually grows.

Design target: cross-session productivity gain — mean steps / token cost for a given workflow drops ≥ 30 % from turn N to turn N+20 — through skill extraction and cache-aware skill loading. Zero token-bill inflation as the library grows.

What Mentat-Learn is

Your personal agent, coherent across every channel — Slack, iMessage, WhatsApp, Telegram, Signal, email, web — backed by a closed skill-learning loop. After you finish a task, Mentat extracts a reusable procedure, writes it as a SKILL.md, refines it on every reuse, and compresses co-occurring skills into higher-order composites. Memory is opt-in and per-channel; you can say "forget that" at any moment.

Niche

One-user or household personal assistant; cross-channel continuity; gets better through use.

Not

Workplace bot (Syndicate); voice-first (Harmony-Voice); ops automation (Aegis-Ops).

Status

Novel contributions

1. Three-way composition: SOUL.md × Hermes × Honcho

Persona anchor (who the agent is) × skill extractor (what it can do) × dialectic user modeling (who the user is to the agent). Each novel alone; the three-way fusion is genuinely new.

2. Skill compression

Co-occurring skills → higher-order composite; base skills preserved as callable but "subsumed". Extends Voyager's skill library idea with a novel compression operator.

3. Cache-aware skill loading

Skills loaded JIT via description match; token bill flat as library grows. Applies Hermes's cache discipline.

4. Autogenesis-versioned skill artifacts

Each skill revision is a versioned patch with rollback-on-regression, per the Autogenesis protocol.

5. Natural-Language Agent Harness (NLAH) slot

Advanced users can supply a HARNESS.md natural-language spec (arXiv:2603.25723) executed by an Intelligent Harness Runtime — per-user harness personalization without forking.

Why this stack

Why a four-layer memory?

Three layers (procedural / facts / session) is what Hermes defaults to. Adding a fourth — the dialectic — captures the relationship between user and agent over time. Personal agents live or die by feeling personal; facts alone don't deliver that.

Why skill compression?

Skill libraries become discoverability problems past ~100 skills. Co-occurring skills being promoted to composites keeps the discovery surface small without losing the base skills (they remain callable).

Why SOUL.md?

Persona drift over months is a real failure. An immutable partition that the context engine can't compact away is the cheapest control. SemaClaw proved the pattern works.

Why self-eval every 15 tasks?

Continuous eval is expensive; per-task eval ignores slow drift. 15-task cadence is Hermes's sweet spot; good enough to catch drift early without burning budget.
